Acon Scheduler
This asset aims to streamline the process of creang coroune acons in a straighorward manner.
Addionally, it enables seamless communicaon between disnct scripts by employing nocaons to
execute acons. This funconality also extends to the ability to share parameters among scripts.
Features
Nofy: You can nofy and send parameters, you can also do it between dierent scripts.
WaitTill: You can execute an acon aer seconds, frames, bool delegate or nofy.
Interval: You can execute an acon at an interval of seconds, frames, bool delegate or nofy.
UnlWaitTill: You can stop the roune before it ends by using another WaitTill
How To Use
To start invoking the funcons of this asset, you will need to add the AconScheduler dependency.
You can add AconScheduler class as a variable or you may directly employ inheritance, given that the
AconScheduler is ulizing MonoBehaviour.

{

}

{

}

WaitTill
To wait seconds, the rst argument must be int or oat.
To wait frames, the rst argument must be int.
To wait ll a nocaon, you need the nocaon string as the rst argument.
Lambdas can receive parameters and even those with fewer parameters will also be noed, max length
10.
You can also wait ll a condion is true with bool funcon.
WaitTill(3, () => Debug.Log("Hi!"));
FrameWaitTill(200, () => Debug.Log("Hi!"));
DelayNotify(3, "test");
WaitTill("test", () => Debug.Log("Hi!"));
DelayNotify(3, "test", "Jimmy", 80);
WaitTill("test", (name, health) =>
{
Debug.Log("Player: " + name);
Debug.Log("Health: " + health);
});
WaitTill("test", (name) => Debug.Log("Hi! " + name));
private bool test = false;

WaitTill(3, () => test = true);

private bool test = false;

private void Start()
{
WaitTill(3, () => test = true);
WaitTill(TestBool, () => Debug.Log("Hi!"));

Interval
To execute a interval every x seconds, the rst argument must be int or oat.
To execute a interval every x frames, the rst argument must be int.
You can control the interval with bool return, return true will keep the interval, return false will stop it.
Interval also be able to receive nocaons and parameters, rst argument must be nofy string.
You can also make a condion interval.
Interval(1, () => Debug.Log("Hi!"));
FrameInterval(200, () => Debug.Log("Hi!"));
int count = 0;
Interval(1, () =>
{
if (count == 3) return false;
Debug.Log(count);
count++;
return true;
});
Interval(3, () => Notify("test", "Jimmy", 80));
Interval("test", (name, health) =>
{
Debug.Log("Player: " + name);
Debug.Log("Health: " + health);

Interval(TestBool, () => Debug.Log("Hi!"));

IEnumerators
This asset also implements funcons similar to those before but that serve as a awaits for corounes.
You can also do mulple WaitTills where you will wait unl one completes.
To obtain the parameters of nofy you must use the Parameters class, using WaitTillParameters. This
class has two methods, WaitTill which will serve as await, and Values an array which will store the
parameters.
private IEnumerator Sample()
{
yield return WaitTill(3);
Debug.Log("Hi!");
yield return FrameWaitTill(200);
Debug.Log("Hi!");
DelayNotify(3, "test");
yield return WaitTill("test");
Debug.Log("Hi!");
DelayNotify(3, "level_end");
yield return WaitTill("death", "level_end");
Debug.Log("Hi!");
test = false;
WaitTill(3, () => test = true);
yield return WaitTill(TestBool);
Debug.Log("Hi!");
}
private IEnumerator Sample()
{
DelayNotify(3, "test");
yield return WaitTill(WaitTill(5), WaitTill("test"));
Debug.Log("Hi!");
}
private IEnumerator Sample()
{
DelayNotify(3, "test", "Jimmy", 80);
Parameters parameters = WaitTillParameters("test");
yield return parameters.WaitTill;
Debug.Log("Player: " + parameters.Values[0]);
Debug.Log("Health: " + parameters.Values[1]);
}
